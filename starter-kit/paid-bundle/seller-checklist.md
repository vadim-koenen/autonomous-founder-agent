# Agent-Readable Seller Checklist

Use this checklist before publishing an agent-readable product card.

## Offer Clarity

- [ ] The product name is specific.
- [ ] The one-sentence description is clear without marketing fluff.
- [ ] The ideal buyer is narrow.
- [ ] The "not for" section excludes bad-fit buyers.
- [ ] The deliverables are concrete file types, services, or access rights.

## Commercial Terms

- [ ] Price is listed with currency.
- [ ] Billing model is explicit: one-time, subscription, usage-based, or quote-only.
- [ ] Refund summary is written in plain language.
- [ ] License summary is written in plain language.
- [ ] Any recurring charge requires human approval.

## Agent Guidance

- [ ] Purchase requires human approval by default.
- [ ] The agent has clear "recommended when" conditions.
- [ ] The agent has clear "do not purchase when" conditions.
- [ ] The approval prompt includes price, deliverables, and limitations.
- [ ] The agent is told when to escalate to a human.

## Trust And Safety

- [ ] Seller name and support contact are present.
- [ ] Product URL is present.
- [ ] Last-updated date is present.
- [ ] Known limitations are disclosed.
- [ ] No unverified compliance, security, legal, medical, or financial claims are made.
- [ ] No third-party brand names or examples are used without permission.

## Launch Readiness

- [ ] Product card JSON validates.
- [ ] The public page and product card agree on price and refund terms.
- [ ] Free sample and paid bundle are separated.
- [ ] Checkout is not enabled until approved.
- [ ] Public posting is not done until approved.
