# Public-Safe Intake

The public intake is for initial qualification only.

Safe to provide:

- public product or repository URL
- one-sentence description of the agent's intended outcome
- current launch stage
- public framework or protocol names
- preferred deliverable format

Do not provide:

- passwords, API keys, tokens, private keys, seed phrases, or wallet credentials
- private repository links or access tokens
- customer personal information
- unpublished source code
- regulated or confidential records
- production database credentials

If the sprint reaches a verified purchase, any private access must be granted through an owner-approved customer system with the narrowest necessary permissions.
